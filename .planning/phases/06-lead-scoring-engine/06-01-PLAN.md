  <files>src/lib/scoring-config.ts, src/lib/scoring.ts</files>
  <action>
Create `src/lib/scoring-config.ts` with:

1. **SIGNAL_WEIGHTS** constant (as const): budget: 35, timeline: 25, company: 15, message: 15, service: 10. Add inline comment "Must sum to 100" and a comment per field explaining what it scores.

2. **TIER_THRESHOLDS** constant (as const): warm: 31, hot: 61. Comment: scores below warm threshold are cold.

3. **LeadTier** type: `'cold' | 'warm' | 'hot'`

4. **Score value maps** (as const Records) — these define the contract Phase 8's form must emit:
   - `BUDGET_SCORES`: `{ under_5k: 0, '5k_10k': 10, '10k_25k': 20, '25k_50k': 28, '50k_plus': 35 }`
   - `TIMELINE_SCORES`: `{ not_sure: 0, '6_months_plus': 5, '3_6_months': 15, '1_3_months': 22, asap: 25 }`
   - `SERVICE_SCORES`: `{ analytics: 6, devops: 8, web_dev: 10 }` (web_dev highest — largest typical deal size)
   - `MESSAGE_THRESHOLDS`: array of `{ minLength: number, points: number }` sorted descending: `[{ minLength: 300, points: 15 }, { minLength: 150, points: 10 }, { minLength: 50, points: 5 }]`

Export all constants and the LeadTier type. This file has ZERO imports — it is the bottom of the dependency chain.

Create `src/lib/scoring.ts` with:

1. **LeadPayload** interface: `{ budget?: string | null; timeline?: string | null; company?: string | null; message: string; serviceType?: string | null; }`

2. **ScoreResult** interface: `{ score: number; tier: LeadTier; }`

3. **scoreLead(payload: LeadPayload): ScoreResult** — sums points from 5 private helper functions, clamps to [0, 100] via `Math.max(0, Math.min(100, Math.round(total)))`, returns `{ score, tier: scoreToTier(score) }`.

4. **scoreToTier(score: number): LeadTier** — exported. Returns 'hot' if >= TIER_THRESHOLDS.hot, 'warm' if >= TIER_THRESHOLDS.warm, else 'cold'.

5. Private helper functions (not exported):
   - `scoreBudget(budget)`: lookup in BUDGET_SCORES, return 0 if null/undefined/unrecognized
   - `scoreTimeline(timeline)`: lookup in TIMELINE_SCORES, return 0 if null/undefined/unrecognized
   - `scoreService(serviceType)`: lookup in SERVICE_SCORES, return 0 if null/undefined/unrecognized
   - `scoreCompany(company)`: returns SIGNAL_WEIGHTS.company if string is non-empty after trim, else 0
   - `scoreMessage(message)`: iterates MESSAGE_THRESHOLDS (sorted desc), returns points for first threshold where message.trim().length >= minLength, else 0

IMPORTANT: `scoreLead()` does NO validation — the contact API handles that upstream. Missing optional fields = 0 points, no redistribution.

IMPORTANT: Do NOT write to the database in this phase. `scoreLead()` is a pure function. Phase 7 wires it into the contact API.
