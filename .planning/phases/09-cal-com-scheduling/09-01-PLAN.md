  <action>
Create `src/pages/api/cal-webhook.ts` as a POST endpoint that:

1. **Reads raw body first** (before any JSON parsing): `const rawBody = await request.text();`

2. **Verifies HMAC-SHA256 signature:**
   - Get secret from `import.meta.env.CAL_WEBHOOK_SECRET`
   - If secret is missing, log error and return `new Response(null, { status: 500 })`
   - Get signature from header `x-cal-signature-256`
   - Compute HMAC: `createHmac('sha256', secret).update(rawBody).digest('hex')`
   - Compare with `timingSafeEqual` wrapped in try/catch (Buffer length mismatch throws)
   - Return `new Response(null, { status: 401 })` on mismatch or error

3. **Parses payload and filters event type:**
   - `JSON.parse(rawBody)` after signature is verified
   - Type the payload as:
     ```typescript
     { triggerEvent: string; payload: { uid: string; startTime: string; attendees: Array<{ email: string; name: string }> } }
     ```
   - If `triggerEvent !== 'BOOKING_CREATED'`, return 200 (acknowledge silently)
   - Extract `bookingUid = event.payload.uid`, `scheduledAt = event.payload.startTime`, `attendeeEmail = event.payload.attendees?.[0]?.email`
   - If no attendeeEmail, log warning and return 200

4. **Matches lead by email (most recent first):**
   - Import `desc` from `drizzle-orm` for ordering
   - Query: `db.select({ id: submissions.id }).from(submissions).where(eq(submissions.email, attendeeEmail)).orderBy(desc(submissions.id)).limit(1).get()`
   - If no match found, log warning `Cal.com webhook: no lead found for email ${attendeeEmail}` and return 200

5. **Updates lead record:**
   - `db.update(submissions).set({ calBookingUid: bookingUid, calScheduledAt: scheduledAt }).where(eq(submissions.id, existing.id)).run()`
   - Return `new Response(null, { status: 200 })`
