## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding, card inner padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Success panel padding: 48px vertical / 32px horizontal (`padding: 3rem 2rem` — pre-existing, source: contact.astro `.success-panel`)
- Cal.com embed container: `min-height: 500px` — dictated by Cal.com iframe content, not the 8-point scale
- Touch targets for Cal.com time slot buttons: minimum 44px height — accessibility requirement for embedded iframe (handled by Cal.com internally)

## Component Inventory

New and modified components for this phase:

### 1. Cal.com Embed Container (new)

**Element:** `<div id="cal-embed-container">` inside the success panel
**Placement:** Replaces the `<a href="${CAL_URL}">Book a Discovery Call</a>` link in the success panel innerHTML
**Sizing:**
- Width: `100%` (fills success panel column)
- Min-height: `500px` (accommodates Cal.com booking calendar iframe)
- Max-width: none — inherits success panel container width
- On mobile (< 600px): `min-height: 600px` to accommodate scrollable slots in portrait orientation

**Visibility rule:** Rendered ONLY after successful form submission. Never present on initial page load or after navigating away and returning (sessionStorage cleared on submission).

**Loading state:** Show a CSS spinner (`.loading-spinner` — pre-existing in style.css) centered in the embed container while Cal.com script loads. Remove spinner once `Cal.loaded` is truthy or `embed.js` fires first render. Source: CONTEXT.md Claude's Discretion — "whether to show a loading spinner."

### 2. Success Panel (modified)

**Pre-existing:** `.success-panel` glass panel with checkmark icon, heading, paragraph.
**Change:** The `.btn.btn-primary.btn-lg` CTA link is removed. The `<div id="cal-embed-container">` is inserted in its place. The heading "Message Sent!" and body paragraph "We'll get back to you within 24 hours." are retained above the embed.

**CSS addition to contact.astro scoped styles:**

```css
#cal-embed-container {
  width: 100%;
  min-height: 500px;
  margin-top: 1.5rem;     /* 24px — matches pre-existing .success-panel .btn margin-top */
  border-radius: 12px;    /* matches .btn border-radius */
  overflow: hidden;       /* clips Cal.com iframe to rounded corners */
  position: relative;
}

#cal-embed-loading {
  position: absolute;
  inset: 0;
  display: flex;
  align-items: center;
  justify-content: center;
}

@media (max-width: 600px) {
  #cal-embed-container {
    min-height: 600px;
  }
}
```

### 3. Webhook Endpoint (backend — no UI surface)

`POST /api/cal-webhook` — no UI component. Returns HTTP 200 with empty body on success. No user-visible state.

## Interaction States

| Surface | State | Visual Treatment |
|---------|-------|-----------------|
| Cal.com embed container | Loading (script injecting) | `.loading-spinner` centered, 16px white spinner at 0.6 opacity |
| Cal.com embed container | Loaded | Spinner removed, Cal.com iframe visible |
| Cal.com embed container | Load error (script fails) | Show fallback link: `<a href="https://cal.com/synctexts/discovery" target="_blank" class="btn btn-primary btn-lg">Book a Discovery Call</a>` — same as pre-Phase-9 behavior |
| View Transition (navigate away then back) | Fresh form state | Embed container not present — no action needed |
| View Transition (navigate away then back) | Success panel state | Not possible — sessionStorage cleared on submission means success panel never shows on re-navigation |

Source: CONTEXT.md View Transitions & Lifecycle + RESEARCH.md Pitfalls 1 and 2.

## Interaction Transitions

All transitions follow the pre-existing pattern:

```css
/* Pre-existing in style.css */
--transition-smooth: all 0.5s cubic-bezier(0.16, 1, 0.3, 1);
--transition-fast: all 0.2s ease;
```

```css
/* Pre-existing in contact.astro scoped styles */
@media (prefers-reduced-motion: no-preference) {
  /* transitions gated — instant fallback for reduced-motion users */
}
```

New in Phase 9: The embed container fade-in (opacity 0 → 1) when Cal.com finishes loading:
- Duration: 0.3s ease
- Gated on `prefers-reduced-motion: no-preference`
- Fallback: instant visible

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Cal.com embed iframe title | Cal.com sets `<iframe title="Cal.com Booking">` internally — verified in embed-core source |
| Loading spinner ARIA | `<div id="cal-embed-loading" role="status" aria-label="Loading booking calendar">` |
| Embed container focus | `#cal-embed-container` is not focusable — focus enters iframe normally via tab |
| Reduced motion | All authored transitions gated on `prefers-reduced-motion: no-preference` (pre-existing pattern from Phase 8) |
| Color contrast | `--text-primary` (#ffffff) on `--bg-base` (#060608) — passes WCAG AA at all declared font sizes |
