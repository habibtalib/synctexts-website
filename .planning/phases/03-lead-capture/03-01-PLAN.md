  <action>
    1. Install dependencies:
       ```
       npx astro add node
       npm install resend drizzle-orm better-sqlite3
       npm install -D drizzle-kit @types/better-sqlite3
       ```

    2. Update `astro.config.mjs`: Add `import node from '@astrojs/node'` and add `adapter: node({ mode: 'standalone' })` to the config. Keep existing site and markdown config. Do NOT add `output: 'hybrid'` -- Astro 5 uses per-page `export const prerender = false` instead.

    3. Create `drizzle.config.ts` at project root:
       - Use `better-sqlite3` driver
       - Schema path: `./src/db/schema.ts`
       - DB file: `./data/submissions.db`
       - Out directory: `./drizzle` for migrations

    4. Create `src/db/schema.ts`:
       - Define `submissions` table using `sqliteTable` with columns:
         - `id`: integer primary key autoincrement
         - `name`: text not null
         - `email`: text not null
         - `company`: text (nullable)
         - `message`: text not null
         - `ip`: text (nullable)
         - `createdAt`: text not null with $defaultFn returning new Date().toISOString()
         - `read`: integer boolean mode, not null, default false
         - `rateLimited`: integer boolean mode, not null, default false

    5. Create `src/db/index.ts`:
       - Import Database from better-sqlite3, drizzle from drizzle-orm/better-sqlite3, and schema
       - Resolve DB path to `data/submissions.db` relative to project root (use `import.meta.dirname` or `process.cwd()` + path.join)
       - Ensure `data/` directory exists (use `mkdirSync` with recursive)
       - Create singleton: `const sqlite = new Database(dbPath)` then `export const db = drizzle({ client: sqlite, schema })`

    6. Create `src/lib/validation.ts`:
       - Export `validateContact(data: Record<string, unknown>): { valid: boolean; errors: Record<string, string> }`
       - Name: required, string, trimmed length >= 2 chars. Error: "Name must be at least 2 characters"
       - Email: required, string, matches /^[^\s@]+@[^\s@]+\.[^\s@]+$/. Error: "Please enter a valid email address"
       - Message: required, string, trimmed length >= 10 chars. Error: "Message must be at least 10 characters"
       - Company: optional, no validation needed

    7. Create `src/lib/rate-limiter.ts`:
       - Export `isRateLimited(ip: string, maxPerHour = 5): boolean`
       - Use Map<string, number[]> to store timestamps per IP
       - Filter timestamps to only those within last hour (3600000ms)
       - Push current timestamp, update map
       - Return true if count exceeds maxPerHour
       - Add cleanup: every 100 calls, iterate map and remove entries with no recent timestamps to prevent memory leak

    8. Create `src/lib/email.ts`:
       - Export `sendContactNotification(data: { name: string; email: string; company?: string; message: string }): Promise<void>`
       - Initialize Resend with `import.meta.env.RESEND_API_KEY`
       - Send email to `import.meta.env.CONTACT_EMAIL`
       - From: `import.meta.env.EMAIL_FROM || 'SyncTexts <onboarding@resend.dev>'`
       - Reply-to: submitter's email
       - Subject: "New inquiry from {name}" (append company in parens if provided)
       - HTML body: clean template with all form fields. Use inline CSS for email compatibility.
       - Throw on error with descriptive message

    9. Create `.env.example` with all required env vars (RESEND_API_KEY, CONTACT_EMAIL, EMAIL_FROM, ADMIN_USER, ADMIN_PASS) with placeholder values and comments.

    10. Update `.gitignore`: Add `data/` directory (SQLite database) and `.env` if not already present.

    11. Run `npx drizzle-kit push` to create the initial database schema.

       - If rate-limited: skip email, still return 200 with success but include a friendly message like "Thank you for your message. We'll be in touch soon." (same as non-rate-limited so user doesn't know they're rate-limited)
       - On success: return 200 with `{ success: true, message: "Thank you! We'll get back to you soon." }`
       - On unexpected error: return 500 with `{ success: false, message: "Something went wrong. Please try again later." }`

       - Add `<span class="field-error" id="{field}-error"></span>` after each input for inline errors

         j. In finally: re-enable button, restore "Send Message" text (unless success replaced the form)
