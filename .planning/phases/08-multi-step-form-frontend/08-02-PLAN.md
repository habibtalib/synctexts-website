**3. State management functions:**
- `defaultState(): FormState` â€” returns `{ step: 1, service_type: null, budget: null, timeline: null, name: '', email: '', company: '', message: '' }`
- `loadState(): FormState` â€” reads `sessionStorage.getItem(STORAGE_KEY)`, wraps `JSON.parse` in try/catch, falls back to `defaultState()`
- `saveState(state: FormState): void` â€” `sessionStorage.setItem(STORAGE_KEY, JSON.stringify(state))`, wrapped in try/catch (silent fail)
- `clearState(): void` â€” `sessionStorage.removeItem(STORAGE_KEY)`

**4. Validation functions (preserve existing signatures):**
- `validateField(field: string, value: string): string | null` â€” exact same logic as current: name >= 2 chars, email regex, message >= 10 chars

- `showErrorSummary(errors: Record<string, string>): void` â€” same as current but heading text is `"Please fix the following:"`
- `clearErrorSummary(): void` â€” same as current

**5. Step indicator update:**
- `updateIndicator(currentStep: number, completedSteps: Set<number>): void`
  - For each step 1-3: query `[data-indicator-step="${i}"]`
  - Toggle classes: `.is-current` (i === currentStep), `.is-complete` (completedSteps.has(i) && i !== currentStep), `.is-pending` (otherwise)
  - Set `aria-current="step"` ONLY on current dot; remove from all others
  - For completed dots: set `role="button"`, `tabindex="0"`; for others: remove those attributes
  - For connecting lines 1-2: query `[data-indicator-line="${i}"]`, toggle `.is-filled` when currentStep > i OR completedSteps has steps beyond that line

**6. Step transition function:**
- `showStep(targetStep: number, direction: 'forward' | 'back'): void`
  - Get current active panel: `.form-step.is-active`
  - Get target panel: `[data-step="${targetStep}"]`
  - If both exist and are different:
    - Check `prefers-reduced-motion`: `window.matchMedia('(prefers-reduced-motion: reduce)').matches`
    - If reduced motion: simply remove `.is-active` from current, add `.is-active` to target (no animation classes)
    - If motion OK:
      - Add entry direction class to target (`enter-from-right` for forward, `enter-from-left` for back)
      - Add `.is-active` to target
      - Force reflow: `target.getBoundingClientRect()`
      - Remove entry direction class from target (starts animation to default position)
      - Add exit class to current (`exit-to-left` for forward, `exit-to-right` for back)
      - Add `transitionend` listener on current with `{ once: true }`: filter `e.propertyName === 'transform'`, then remove `.is-active`, `.exit-to-left`, `.exit-to-right` from current

**7. Form field sync (sessionStorage live write):**
- `syncFieldsToState(state: FormState): void` â€” reads current DOM values for budget, timeline, name, email, company, message into state object. Called before every `saveState()` during navigation.
- `hydrateFieldsFromState(state: FormState): void` â€” writes state values back to DOM inputs/selects. Called during initialization from sessionStorage.
