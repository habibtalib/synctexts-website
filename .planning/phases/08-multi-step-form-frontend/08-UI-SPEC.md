## Animation / Transition Contract

### Step Slide Transitions

| Property | Value |
|----------|-------|
| Mechanism | CSS `transform: translateX` + `opacity` via class swap |
| Duration | `300ms` |
| Easing | `ease` |
| Enter offset | `40px` (right for forward, `-40px` for back) |
| Exit offset | `-40px` (left for forward, `40px` for back) |
| Entry opacity start | `0` |
| Exit opacity end | `0` |
| `prefers-reduced-motion` fallback | All transition/animation properties set to `0ms` or `none`; steps appear/disappear instantly |

CSS implementation (media-query-gated):

```css
@media (prefers-reduced-motion: no-preference) {
  .form-step {
    transition: transform 0.3s ease, opacity 0.3s ease;
  }
  .form-step.enter-from-right { transform: translateX(40px); opacity: 0; }
  .form-step.enter-from-left  { transform: translateX(-40px); opacity: 0; }
  .form-step.exit-to-left     { transform: translateX(-40px); opacity: 0; }
  .form-step.exit-to-right    { transform: translateX(40px); opacity: 0; }
}
```

`transitionend` listener on exiting step uses `{ once: true }` and filters on `e.propertyName === 'transform'` to avoid double-fire.

### Connecting Line Fill Animation

| Property | Value |
|----------|-------|
| Mechanism | CSS `width` transition on `::after` pseudo-element, or class swap on `.step-line.is-filled` |
| Duration | `300ms` |
| Easing | `ease` |
| Sync | Starts simultaneously with step slide |
| `prefers-reduced-motion` | Instant class swap, no transition |

### Service Card Hover

| Property | Value |
|----------|-------|
| Mechanism | CSS transition on `transform`, `border-color`, `box-shadow` |
| Duration | `0.5s cubic-bezier(0.16, 1, 0.3, 1)` (matches `--transition-smooth`) |
| Scale | `1.02` on hover/selected |
| Selection flash | On click: `scale(0.98)` for `80ms` then `scale(1.02)` before auto-advancing (subtle tap feedback) |

### Success Panel Entry

| Property | Value |
|----------|-------|
| Mechanism | `fadeIn` keyframe (already in `global.css`) |
| Duration | `0.5s ease-out` |
| Keyframes | `opacity: 0, translateY(10px)` → `opacity: 1, translateY(0)` |

### Hydration on `astro:page-load`

- No animation classes applied during sessionStorage hydration
- Only `is-active` class set directly — no `enter-from-*` classes
- Prevents flicker on page reload to mid-form state

## Layout Contract

### Step Indicator Placement

Place the step indicator **inside the form column** (right-side of `.contact-wrapper`), not spanning both columns. This matches the Stripe/Typeform pattern — the indicator is part of the form area, not a page-level header.

Structure within the right column:
```
.step-indicator   (full width of form column)
.step-labels      (full width of form column, aria-hidden)
.form-steps-container (overflow: hidden; contains all .form-step panels)
```

### Service Cards Grid

```css
.service-select-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}
```

At `≤ 600px`: `grid-template-columns: 1fr` (stacked).

### `.contact-wrapper` Responsive Behavior

- `≥ 900px`: `grid-template-columns: 1fr 1fr`, `padding: 4rem`, `gap: 4rem`
- `< 900px`: `grid-template-columns: 1fr`, `padding: 2.5rem`, `gap: 2.5rem` (existing breakpoint unchanged)

Step indicator and form steps sit inside the right grid cell on desktop; they become full-width on mobile since the grid collapses to 1 column.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Step indicator semantics | `aria-label="Form progress"` on `.step-indicator`; each dot has `aria-label="Step N: Label"` |
| Current step announcement | `aria-current="step"` on the active `.step-dot` |
| Completed dot keyboard activation | `role="button"`, `tabindex="0"`, `onkeydown` handler for Enter/Space |
| Error announcements | `.form-errors` has `role="alert"` (existing); focus moves to first `.input-error` field on validation failure |
| Reduced motion | All transitions gated on `@media (prefers-reduced-motion: no-preference)` |
| Select styling | `color-scheme: dark` preserves OS-level accessible text contrast in option dropdowns |
| Service cards | `type="button"` elements (not `div`s) — natively focusable, keyboard-activatable |
| Honeypot field | `aria-hidden="true"`, `tabindex="-1"` (existing, preserved) |
