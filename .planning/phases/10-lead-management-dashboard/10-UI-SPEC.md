## Spacing Scale

Declared values (multiples of 4 only). Pre-populated from `style.css` existing patterns and `index.astro` admin page measurements.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, note-icon inline padding, letter-spacing |
| sm | 8px | Badge internal padding (`0.2rem 0.6rem`), button gaps, action row gaps |
| md | 16px | Default element spacing, filter toolbar internal gap |
| lg | 24px | Compact row vertical padding, toolbar-to-list gap |
| xl | 32px | Admin section internal padding |
| 2xl | 48px | Section padding top/bottom |
| 3xl | 64px | Not used in admin context |

Exceptions:
- Compact row height: minimum 44px touch target for row click (accessibility requirement)
- Expanded panel max-height ceiling: 600px (CSS max-height transition ceiling — not a spacing token)
- Container padding: 2rem (32px) horizontal — inherited from `.container` in `style.css`
- Card border-radius: 16px — inherited from `.submission-card` pattern
- Compact row gap and padding: `gap: 12px` and `padding: 12px 16px` — 12px is not a named scale token. Used here because the compact row requires tighter-than-md (16px) density to fit name, email, badges, and date in a single scannable line without crowding. 16px gap makes the row feel too wide; 8px is too tight for visual separation between interactive badge clusters. 12px is the midpoint that achieves linear.app-style row density. Exception applies to `.lead-row-compact` only.

## Typography

Source: `style.css` `:root` and existing `index.astro` admin CSS patterns.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 16px (1rem) | 400 (regular) | 1.6 | Inter | Lead message text, notes textarea, expanded panel content |
| Label | 13.6px (0.85rem) | 400 (regular) | 1.4 | Inter | Submission date, IP address, secondary meta, pagination buttons, notes save button, save confirmation flash |
| Badge | 11.2px (0.7rem) | 600 (semibold) | 1 | Inter | All badges (score, status, read, HubSpot, booked, rate-limited) |
| Heading | 20px (1.25rem) | 600 (semibold) | 1.2 | Outfit | Lead name in compact row |

Typography rules:
- Badges: `text-transform: uppercase; letter-spacing: 0.5px` — matches existing `.badge-score` pattern
- Compact row name: `font-weight: 600` in Outfit (display font) to anchor the row visually
- Notes textarea: 16px Inter at weight 400 — matches `.glass-input` body text from Phase 8
- Pagination label ("Page 2 of 5", "Showing 26-50 of 127 leads"): 13.6px Inter at weight 400, color `--text-dimmed`
- Filter toolbar labels: 13.6px Inter at weight 400, color `--text-secondary`
- Pagination buttons (`.btn-pagination`): 13.6px (0.85rem) Inter — buttons at label scale is standard; aligns with pagination info text in the same control row
- Notes save button (`.notes-save-btn`) and save confirmation (`.notes-save-confirm`): 13.6px (0.85rem) Inter — consistent with label scale used throughout the expanded panel

## Interaction Contract

### Compact Row Expand/Collapse

- **Trigger:** Click anywhere on `.lead-row-compact` except `button`, `a`, `select`, `textarea` children
- **Mechanism:** Toggle `.expanded` class on `.lead-row`; CSS `max-height` transition on `.lead-panel`
- **Guard:** `if (e.target.closest('button, a, select, textarea')) return;`
- **Motion:** `transition: max-height 0.3s ease` gated on `@media (prefers-reduced-motion: no-preference)`. No transition fallback — instant show/hide.
- **Ceiling:** `max-height: 600px` on `.lead-row.expanded .lead-panel`
- **Multiple open:** Allowed — no accordion constraint. Admin may compare two leads simultaneously.

### Status Dropdown (AJAX)

- **Location:** Expanded panel only — not visible in compact row
- **Trigger:** `change` event on `.status-select`
- **Endpoint:** `POST /api/admin/update-status` — body `{ id: number, status: string }`
- **Success:** Update `.badge-status` class and text in compact row instantly (no page reload)
- **401 response:** `window.location.reload()` to re-trigger Basic Auth
- **Failure:** Revert select to previous value; no visual error (silent fail — admin can retry)

### Filter Form (URL navigation)

- **Mechanism:** `<form method="get">` with `<select name="status">`, `<select name="service">`, `<select name="min_score">`, `<select name="sort">` elements
- **Apply:** Form submit button OR `select.addEventListener('change', () => form.submit())` for instant application
- **Page reset:** `<input type="hidden" name="page" value="1">` inside form — always resets to page 1 on filter change
- **Active state:** On page load, `<select>` options with value matching URL param receive `selected` attribute (server-rendered). Active selects get `.active` CSS class added via JS on `astro:page-load`.
- **Clear filters:** "Clear filters" link navigates to `/admin` (no params)

### Pagination

- **Previous button:** `<a href="{prevUrl}">` — disabled (visually) when `pageParam === 1`; `prevUrl = null` renders as `<button disabled>`
- **Next button:** `<a href="{nextUrl}">` — disabled when `pageParam === totalPages`; `nextUrl = null` renders as `<button disabled>`
- **No JS required** — pure server-side URL navigation

### Notes Save (AJAX)

- **Trigger:** Explicit click on "Save Note" button (no auto-save, no keyup save)
- **Endpoint:** `POST /api/admin/update-notes` — body `{ id: number, notes: string }`
- **Success:** Flash `.notes-save-confirm` "Saved" text for 1.5s then fade to opacity 0; update `.note-icon` visibility in compact row
- **401 response:** `window.location.reload()`
- **Failure:** Change button text to "Save failed" for 2s then revert to "Save Note"

### Toggle Read / HubSpot Sync

- Preserved unchanged from existing implementation in `index.astro` — no modifications

## Layout Contract

### Page Structure

```
<BaseLayout>
  <PageHeader title="Lead Dashboard" subtitle="Manage and qualify leads" />
  <section class="container admin-section">
    [Filter Toolbar]          — glass-panel, full width
    [Results Summary]         — "Showing 26-50 of 127 leads", --text-dimmed, 0.85rem

      [Lead Row × N]          — each is glass-panel + compact row + hidden expanded panel
    [Pagination Controls]     — Previous / Page N of M / Next
  </section>
</BaseLayout>
```

### Compact Row Layout

```
.lead-row-compact (flex, align-center, gap: 12px, min-height: 44px, padding: 12px 16px, cursor: pointer)
  [indigo left border if unread — 3px solid --primary]
  [name]           — flex: 0 0 160px; Outfit 1.0rem weight 600; truncate with ellipsis
  [email]          — flex: 1 1 200px; Inter 0.85rem --text-secondary; truncate
  [score badge]    — flex: 0 0 auto; badge-score pattern
  [status badge]   — flex: 0 0 auto; badge-status pattern
  [service badge]  — flex: 0 0 auto; 0.7rem uppercase; --text-dimmed glass style
  [booked badge]   — flex: 0 0 auto; only if cal_booking_uid is set
  [note icon]      — flex: 0 0 auto; only if notes is non-null/non-empty; title="Has note"
  [date]           — flex: 0 0 auto; 0.85rem --text-dimmed; margin-left: auto
```

Mobile (< 600px): name + email stack vertically; badges wrap; date hidden (too narrow).

### Expanded Panel Layout (inside `.lead-panel`)

```
.lead-panel-inner (padding: 0 16px 16px 16px)
  [Divider]                  — 1px border-top rgba(255,255,255,0.08)
  [Section: Message]         — .lead-panel-label "Message", then <p> with message text
  [Section: Details]         — grid 2-col: Budget | Timeline, Service Type | Company
  [Section: Cal.com Booking] — only if cal_booking_uid; shows "Booked" label + Scheduled time + Booking ID
  [Section: Notes]           — .lead-panel-label "Notes", notes-textarea pre-filled, Save Note button + flash
  [Section: Status]          — .lead-panel-label "Status", status-select dropdown
  [Section: Actions]         — flex row: "Mark as Read/Unread" btn + HubSpot badge/sync btn/link + IP address
```
