  <action>
1. Create `src/pages/api/admin/hubspot-sync.ts`:
   - Mirror the structure of `toggle-read.ts` exactly (prerender=false, POST handler, auth guard, JSON parse, error responses).
   - Import `checkBasicAuth` from `../../../lib/auth`, `syncToHubSpot` from `../../../lib/hubspot`, db and schema from `../../../db`.
   - Auth guard: return 401 with WWW-Authenticate header if `!checkBasicAuth(request)`.
   - Parse body, require `body.id` as number. Return 400 if missing/invalid.
   - Query submission by ID. Return 404 if not found.
   - Call `await syncToHubSpot({...})` (AWAIT here, not fire-and-forget -- admin wants to see the result).
   - After sync, re-query the submission to get the updated `hubspotId`.
   - On success: return `{ success: true, hubspotId: updated.hubspotId }` with status 200.
   - On error (sync throws): `console.error`, return `{ success: false, error: 'Sync failed' }` with status 500.

2. Update `src/pages/admin/index.astro`:

   a. In the frontmatter, read `HUBSPOT_PORTAL_ID`:
      ```
      const hubspotPortalId = import.meta.env.HUBSPOT_PORTAL_ID || '';
      ```

   b. In each submission card's `.submission-header`, after the score badge and before the "Mark as Read" button, add HubSpot sync status:
      - If `sub.hubspotId` is truthy:
        - Show green "Synced" badge: `<span class="badge-hubspot-synced">Synced</span>`
        - Show "View in HubSpot" link (only if `hubspotPortalId` is set):
          `<a href={`https://app.hubspot.com/contacts/${hubspotPortalId}/contact/${sub.hubspotId}`} target="_blank" rel="noopener" class="hubspot-link">View in HubSpot</a>`
      - If `sub.hubspotId` is falsy:
        - Show grey "Not Synced" badge: `<span class="badge-hubspot-unsynced">Not Synced</span>`
        - Show "Sync to HubSpot" button:
          `<button class="btn btn-hubspot-sync" data-id={sub.id}>Sync to HubSpot</button>`

   c. Add CSS for new elements (in the existing `<style>` block):
      - `.badge-hubspot-synced`: green badge matching `.badge-score-hot` pattern (rgba(16,185,129,0.15) bg, #10b981 color, same border/radius/font).
      - `.badge-hubspot-unsynced`: grey badge matching `.read-indicator.read` pattern (rgba(255,255,255,0.05) bg, var(--text-dimmed) color).
      - `.btn-hubspot-sync`: same styling as `.btn-toggle-read`.
      - `.hubspot-link`: small link styled with `color: var(--primary); font-size: 0.8rem; text-decoration: underline;`.

   d. Add JavaScript for sync button (in the existing `<script>` block, inside the `astro:page-load` listener):
      - Select all `.btn-hubspot-sync` buttons.
      - On click: get `data-id`, disable button, set text to "Syncing...".
      - POST to `/api/admin/hubspot-sync` with `{ id }` as JSON body.
      - If `res.status === 401`, reload page (triggers auth dialog).
      - On success (`data.success`): replace the button AND the "Not Synced" badge with a "Synced" badge + "View in HubSpot" link (use `hubspotPortalId` passed via a data attribute on a parent element: add `data-portal-id={hubspotPortalId}` on the `.submissions-list` div).
      - On failure: set button text to "Sync failed", re-enable after 2 seconds.
